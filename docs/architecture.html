🍉 水果突击 · Fruit Assault 系统架构文档

1. 项目概览

水果突击 是一款 Canvas 2D 实时塔防/自走棋风格 Web 游戏。玩家在 5×3 棋盘上放置并合成水果士兵，自动出兵攻击敌方城墙。核心玩法融合了 Merge（合成升级）、TD（塔防）和 Auto Battler（自走棋）三种机制。

5. 核心系统详解

5.1 渲染管线 — 函数覆盖链

Canvas 渲染采用"后加载覆盖"模式：多个 JS 文件定义同名函数，最后一个加载的文件胜出。

index.html 加载顺序 (相关部分): 82 js/render.js → draw(), drawBackground(), drawWall() 83 js/skin.js → 覆盖 draw() (含 shake 效果) 109 js/fruit_skin.js → 覆盖 drawBackground() (渐变缓存) 110 js/battle_skin.js → 覆盖 drawWall(), drawField() (纯色城墙) 最终活跃函数: draw() ← skin.js drawBackground() ← fruit_skin.js (带渐变缓存) drawWall() ← battle_skin.js (纯色, 无渐变) drawField() ← battle_skin.js

5.2 果汁经济闭环

SP（果汁）是游戏唯一主动资源。设计原则：操作越多越贵，击杀回本，主动召唤消耗果汁。

┌──────────────────────────────────────────────────┐ │ 果汁循环 │ │ │ │ 被动收入 ──→ SP 池 ←── 击杀返还(按等级) │ │ 5s+1 │ │ │ │ 消耗 │ │ ▼ │ │ 召唤新水果 ← 递增成本(1→2→...→12) │ │ 急派兵 ← 同上成本 │ │ │ │ 敌方对称: 同样的经济规则, 只是被动略快(4s) │ │ │ │ 保底机制: 城墙每掉一定比例, 获得额外 SP │ └──────────────────────────────────────────────────┘

急派兵同上双击已有水果营

8. 架构模式与约定

🔧 Function Override

多个 JS 定义同名函数，靠 script 加载顺序决定最终版本。用于渲染管线和部分经济函数。

🔄 Monkey-patch wrap

核心函数（如 update()）被多个模块通过"保存旧引用→包裹→替换全局"的方式扩展，形成 6 层 wrap 链。

🎯 全局状态

state/meta/ctx 等大全局变量驱动所有逻辑，无模块化导入导出。所有 JS 共享 window 作用域。

🏛️ Shell + Canvas 双 UI

DOM Shell (product_shell.js) 处理菜单/商店/社交等所有非战斗界面。Canvas 仅在战斗中活跃。

🔐 VM 沙箱 PvP

服务端用 Node.js VM 加载 45+ 客户端 JS 文件，vm.Script 运行，stub DOM/Canvas API。保证模拟一致性。

💾 双存储

本地: localStorage 存 meta(进度/碎片/科技)。云端: SQLite 存用户/聊天/PvP 战绩。云存档覆盖本地。
