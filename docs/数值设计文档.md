## 1. 核心战斗公式

### 1.1 攻击 → 伤害流程

```
兵.atk × LEVEL_MUL[兵.level]
  × roleCounterMultiplier(攻方role, 守方role)    ← 4×4 克制矩阵 0.80~1.35
  × 技能加成（首击暴击/点燃等）
  → 原始伤害 raw

armorReduction = 1 - armor / (armor + 50)          ← 线性减伤
实际伤害 = max(1, raw × armorReduction)

护盾吸收 → 扣血
```

**减伤曲线：**
| armor | 减伤率 |
|-------|--------|
| 0 | 0% |
| 5 | 9.1% |
| 10 | 16.7% |
| 14 | 21.9% |
| 16 | 24.2% |
| 18 | 26.5% |

### 1.2 星级倍率（局内合成）

```js
LEVEL_MUL = [0, 1.0, 1.28, 1.62, 2.02, 2.48, 2.98, 3.55]
```

| Lv | 倍率 | 相邻提升 |
|----|------|---------|
| 1 | 1.00 | — |
| 2 | 1.28 | +28% |
| 3 | 1.62 | +27% |
| 4 | 2.02 | +25% |
| 5 | 2.48 | +23% |
| 6 | 2.98 | +20% |
| 7 | 3.55 | +19% |

**Lv7 是 Lv1 的 3.55 倍，提升幅度逐级递减。**

### 1.3 英雄等级（局外养成）

```js
heroMul(lv) = 1 + (lv - 1) × 0.035   // Lv1=1.0, Lv20=1.665
```

| Lv | heroMul | 累计提升 |
|----|---------|---------|
| 1 | 1.000 | — |
| 5 | 1.140 | +14% |
| 10 | 1.315 | +31.5% |
| 15 | 1.490 | +49% |
| 20 | 1.665 | +66.5% |

**养成总提升 66.5%，平滑无断档。** 每级碎片成本递增：

```js
heroFragCost(lv) = 8 + lv × 4 + band × 12   // band = floor((lv-1)/5)
heroGoldCost(lv) = heroFragCost(lv) × 18 + lv × 25
```

| Lv | 碎片 | 金币 | band |
|----|------|------|------|
| 1→2 | 12 | 241 | 0 |
| 5→6 | 32 | 626 | 0 |
| 6→7 | 48 | 939 | 1 |
| 10→11 | 68 | 1349 | 1 |
| 11→12 | 84 | 1687 | 2 |
| 15→16 | 108 | 2209 | 2 |
| 16→17 | 128 | 2619 | 3 |
| 20 | — | — | — |

**每 5 级一个 band，碎片消耗跳变。** Lv5→6 是 32 碎片，Lv6→7 变成 48（+50%）。

## 4. 经济循环

### 4.1 果汁（潮汐能）

```js
TUNING.juice = {
  start: 8,                     // 开局果汁
  enemyStart: 8,                // 敌方开局果汁
  passiveInterval: 5.0,         // 每 5s 自动回 1
  enemyActionInterval: 4.0,     // 敌方每 4s 尝试召唤
  actionCostCurve: [1,1,2,2,3,3,4,4,5,6],  // 召唤成本递增
  urgentCost: 1,                // 双击急派兵成本
  maxActionCost: 12,            // 最大召唤成本
  wallPitySteps: 4,             // 城墙怜悯分 4 档
  wallPityGain: 2,              // 每档补 2 果汁
}
```

**召唤成本曲线（前 10 次）：**
```
1→1→2→2→3→3→4→4→5→6→(此后按次数+1, 上限12)
```

**开局能做的事：**
- 8 果汁 = 前 3 次召唤（1+1+2=4）还剩 4 果汁
- 或 前 4 次召唤（1+1+2+2=6）剩 2
- 或 双击急派兵 8 次（每次 1）

### 4.2 敌方经济

- 起始 8 果汁，每 5s 回 1
- 每 4s AI 尝试召唤一个兵（成本看 `enemySummonCostCounter`，从 1 起步递增）
- ⚠️ 敌方 AI 的召唤成本是独立的，不限制总兵数

### 4.3 城墙怜悯系统（Pity）

```js
// 当玩家墙 HP 低于 (1 - n/5) 时触发，最多 4 次
// 每次补 2 果汁，共最多补 8
```

| 城墙剩余 | 第几次触发 | 累计补果汁 |
|----------|-----------|-----------|
| < 80% | 第 1 次 | +2 |
| < 60% | 第 2 次 | +4 |
| < 40% | 第 3 次 | +6 |
| < 20% | 第 4 次 | +8 |

**这是平衡玩家墙太脆的创可贴。** 设计意图是"墙越低补得越多"，但本质上是因为玩家墙太脆才需要这个机制。

### 4.4 金币经济

```js
rewards: { baseGold: 18, goldPerStage: 8, bossGoldBonus: 24 }
stageReward(k) = 18 + k × 8 + (boss ? 24 : 0)
```

| 关卡 | 金币 | BOSS 额外 |
|------|------|----------|
| 1 | 26 | — |
| 5 | 58 | +24 |
| 10 | 98 | +24 |
| 20 | 178 | +24 |

### 5.2 敌方成长曲线

```
敌方Lv:  1.00 → 3.33 (20关, +233%)
```

### 5.3 动态难度（关外缩放）

```js
// 仅对敌方生效，按关卡等级缩放
stageAtkMul = 1 + (stageLevel - 1) × 0.04   // 关20 = 1 + 2.33×0.04 = 1.093
stageHpMul  = 1 + (stageLevel - 1) × 0.08   // 关20 = 1 + 2.33×0.08 = 1.186
```

这个乘数**很小**——关 20 时敌方只 +9% 攻击和 +19% 血量。与实际关卡数据相比微不足道（墙 HP 涨了 650%）。**可以说这个系统的实际效果被关卡数据淹没了。**

```
MAX_SOLDIERS = 10   // 每方上限 10 个兵
```

### 7.2 攻城伤害公式（近战打墙）

```js
// 玩家方：
base = round((level × 1.45 + atk × 0.105) × siege)
// 敌方方：
base = round((level × 1.25 + atk × 0.075) × siege)

// 协攻加成：每多 1 个排队兵 +8%，最多 3 个 +24%
```

**玩家攻城效率比敌方高约 20%**（系数 1.45 vs 1.25, 0.105 vs 0.075）。这是有意为之，让玩家攻城比敌方快。

### 7.3 远程攻城伤害

```js
base = round((level × 1.25 + atk × 0.085) × siege)
// 比近战攻城公式低约 20%（缺少 level×1.45 中的 0.20 加成）
```

### 7.4 移速公式

```js
fruitMoveSpeed(s, base) = max(base × 0.78, base × (move / 92) × slow)
  // 不低于基准的 78%
  // slow 减速时 ×0.55（冰水母减速）
  // tide 涌潮时移速 +8%
```

实际推进速度（march 模式）：
```
melee:   SIEGE_SPEED(104) × (move/92) × 0.55~1.0
shooter: SIEGE_SPEED(104) × (move/92) × 0.86
```

## 8. 养成系统的乘数叠加

### 8.1 总乘数链

```
兵的实际属性 = 基础值 × LEVEL_MUL[局内Lv] × heroMul(英雄Lv) × 技能加成
```

示例：Lv3 章鱼刺客 + 英雄 Lv10
```
atk = 22 × 1.62 × 1.315 = 46.8
hp  = 24 × 1.62 × 1.315 = 51.1
```

Lv7 章鱼刺客 + 英雄 Lv20（理论最大值）：
```
atk = 22 × 3.55 × 1.665 = 130.0
hp  = 24 × 3.55 × 1.665 = 141.8
```

**最大值 vs 最小值 = 5.9 倍**（3.55 × 1.665）。一个全养成的 Lv7 章鱼刺客是 Lv1 无养成的 5.9 倍强。

### 8.2 星级特效加成（额外）

| 星级 | 效果 |
|------|------|
| ★3 | 技能 CD -0.5s |
| ★5 | 技能强化（Lv7 额外效果） |
| ★6 | 同职责光环 +3% ATK |
| ★7 | 开局 SP+2 |

这些是游戏后期（英雄 Lv8+）才解锁的，对前期影响小。

## 9. 现存问题

### 9.1 玩家墙 vs 怪墙不对称

```
玩家：72 + wallLv × 5     (初始 72)
敌方：120（关1）→ 900（关20）
```

玩家初始墙血只有敌方的 **60%**。虽然用怜悯系统补偿，但**固定值差异导致前期容错率极低**。

### 9.2 关卡血量增长过快

```
关1→20：怪墙 HP +650%
怪Lv：  +233%
玩家养成：+66.5%（英雄 Lv20）
```

如果玩家不养英雄，从关 1 到关 20 敌方 HP 涨了 7.5 倍，而玩家兵只靠星级成长（3.55 倍），**中后期必然出现"打不动墙"的问题。**

### 9.3 动态度系统被关卡数据淹没

`enemyPveStatMultipliers` 只提供 +9% atk / +19% hp（关 20）的缩放，而关卡本身的墙血涨了 650%。**这个系统几乎无实际效果。**

### 9.4 克制矩阵全面改了，但技能关联没动

- `ROLE_COUNTER_MATRIX` 从 7×7 改为 4×4
- 但 `skill_system_v17.js` 的加成系数还是按旧角色体系写的
- `skill_system_v70.js` 的被动技能还在检查旧角色类型
- 部分技能效果现在可能因为角色重分类而不再触发

### 9.5 经济曲线没做验证

召唤成本 `[1,1,2,2,3,3,4,4,5,6]` 没有对应"玩家在 X 秒内能召 Y 次"的验证。同样，敌方 AI 每 4 秒召一个的成本递增也没验证——敌方到底在前 60 秒能出多少兵？这个数字没人算过。
