# 可派发 Agent 任务 Brief

使用说明：每个 agent 开始前必须先读 `docs/prototype-audit.md` 和 `docs/agent-execution-plan.md`。不要假设项目是空白原型。

## Agent 0：回归测试与实现基线

你负责给当前原型建立可重复验证的测试基线，不改玩法。

必须先读：
- `docs/prototype-audit.md`
- `index.html`
- `js/config.js`
- `js/state.js`
- `js/combat.js`
- `js/juice_economy.js`
- `js/lane_block_fix.js`

任务：
- 新增一个无需浏览器的 VM 回归测试脚本。
- 测试至少覆盖：
  - 开局 SP 是 8。
  - 行动成本从 1 开始。
  - 城墙保底：每掉 20% 给 +3SP，最多 4 次。
  - 同路敌兵存在时不会互相错过直接拆墙。
  - 到墙边时如果同路仍有敌兵，会先清线。
- 写清楚运行命令。

验收：
- 测试失败时进程非 0 退出。
- `node --check` 全量通过。
- 不改游戏规则。

## Agent 1：SP 经济收口

你负责把 SP/果汁经济集中到 `js/juice_economy.js`，避免多个补丁重复改。

当前原型已经有：
- 递增行动成本。
- 手动召唤。
- 双击急派。
- 击杀回 SP。
- 敌方 SP。
- PVP 传递行动 cost。

设计目标：
- 初始 SP = 8。
- SP 无上限。
- 每 5 秒 +1。
- 击杀按被击杀单位等级 +SP。
- 召唤和急派共用同一个递增计数器。
- 合成免费，不增加计数器。
- 城墙每掉 20% +3SP，最多 4 次。

任务：
- 在 `js/juice_economy.js` 增加或整理统一函数：
  - `nextJuiceActionCost`
  - `grantJuice`
  - `spendJuiceAction`
  - `resetJuiceEconomyForLevel`
- 检查 `balance_fix_v15.js`、`main.js`、`pvp.js` 是否仍有绕过经济函数的逻辑。
- 保持 PVP 的本地/远端 cost 同步。

验收：
- 手动召唤和急派成本连续递增。
- 合成不影响成本。
- PVP 不报同步异常。
- 有回归测试覆盖。

## Agent 2：五路战斗与清线规则

你负责战斗核心：五路推进，每路独立战线。

当前原型已经有：
- `combat.js` 基础索敌、移动、攻墙。
- `balance_fix_v15.js` 后排站位和职责索敌。
- `lane_block_fix.js` 同路阻塞。
- `combat_pacing_v19.js` 全兵种参与攻城。
- `skill_system_v17.js` 技能会影响移动/击退/攻墙。

任务：
- 梳理最终生效的：
  - `updateSoldier`
  - `findTarget`
  - `attackTarget`
  - `attackWall`
- 保证同路敌兵存在时优先清线。
- 保证后排不会无脑冲到墙根。
- 保证攻城位仍然有效：清线后可拆墙，队列单位可支援拆墙。
- 检查香蕉突进、菠萝击退是否破坏 lane 归属。

验收：
- 同路双方必然接战。
- 有敌兵时不直接拆墙。
- 清线后能拆墙。
- 远程/辅助不会成为最前排常态。

## Agent 3：伤害公式和 7x7 克制矩阵

你负责把伤害计算从简化职责克制，迁移到最终文档的矩阵化规则。

当前原型已经有：
- `roleCounterMultiplier`
- `applyFruitDamage`
- 护甲字段
- 护盾字段
- 橙炮、香蕉、柠檬等特殊伤害逻辑

目标：
- 减伤率 = 护甲 / (护甲 + 50)
- 实际伤害 = ATK × 等级倍率 × 克制倍率 × (1 - 减伤率)
- 最小伤害 = 1
- 角色矩阵：坦克、前排、突击、后排、攻城、控制、辅助

任务：
- 新增矩阵常量。
- 新增统一伤害函数。
- 让 `applyFruitDamage` 和技能额外伤害尽量走统一入口。
- 保留原型手感：橙子攻城强、清兵弱；突击克后排。

验收：
- 所有普通伤害都能追到同一个公式。
- 护甲测试符合公式。
- 默认 5 单位仍能正常打。

## Agent 4：统一状态引擎

你负责把现有散落状态字段统一。

当前原型已有散落字段：
- `slowTimer`
- `slowMul`
- `shield`
- `_v17ArmorBreak`
- `_v17ArmorBreakValue`
- `skillState`

目标状态：
- frozen
- slowed
- burning
- armorBreak
- knockback
- stunned
- invisible

任务：
- 给 soldier 增加 `statusEffects`。
- 实现：
  - `applyStatus`
  - `updateStatusEffects`
  - `canMove`
  - `canAttack`
  - `isTargetable`
- 先兼容旧字段，再逐步迁移技能。

验收：
- 减速、破甲、护盾旧逻辑不坏。
- 冰冻/眩晕能阻止移动攻击。
- 隐身不能被普通索敌锁定。

## Agent 5：单位池扩展

你负责从当前 13 单位扩到最终设计需要的单位池。

当前单位：
- 11 个可战斗/支援单位。
- 2 个合成辅助单位：奇异果、百香果。
- 默认 5 个核心单位已有技能。

任务：
- 先写一张映射表：最终文档单位 -> 当前已有/需新增/需改名。
- 不要删除当前默认 5 技能。
- 第一批优先补经济辅助球。
- 第二批补缺失战斗单位基础属性。
- 第三批等状态引擎稳定后补复杂技能。

验收：
- 新单位能在 lab/shop/deck 出现。
- 默认卡组不变。
- 纯经济/合成辅助不会被敌方随机池错误刷出，除非设计明确允许。

## Agent 6：局外养成整理

你负责整理当前产品壳里的成长系统。

当前原型已有：
- 老科技树：攻击/血量/城墙/SP 泵。
- 产品壳碎片：`shell.fragments`。
- 初始等级：`shell.fruitLv`，最高 4。
- 关卡星级：`meta.stars`。

目标：
- 果实等级可扩展到 Lv20。
- 新增水果星级，不能和关卡星级混用。
- 碎片升级公式按最终设计逐步接入。

任务：
- 新增 `shell.fruitStars` 或等价字段。
- 明确旧科技树是否保留、隐藏或并入实验室。
- 扩展实验室 UI 显示：碎片、等级、星级、上阵状态。

验收：
- 旧存档不崩。
- 关卡星级和水果星级分开。
- 召唤出的己方单位能吃到正确养成加成。

验收：

## Agent 8：PVP 稳定和规则兼容

你负责在不破坏房间码 PVP 的基础上，为最终 PVP 规则做准备。

当前原型已有：
- WebSocket 房间服务。
- 创建/加入/准备/开始。
- 指令同步：召唤、移动、合成/交换、急派。
- 本地双端模拟。
- 断线结果。

任务：
- 检查所有随机源是否在 PVP 可控。
- 给协议加版本或兼容说明。
- 加 PVP 专属数值入口，但不要立刻硬改所有规则。
- 分池匹配先留接口，不替换房间码。

验收：
- 房间码对战仍能跑。
- 同样操作序列保持确定性。
- 断线处理正常。
